The "A" in AJAX stands for "Asynchronous"

Until now, we have studied asynchronism resulting
  1. from DOM events (window.addEventListener("DOMContentLoaded",initFunctionCallBack))
  2. from user interactions (button.addEventListener("click",actionCallBack))
  3. from the call of the Interval API (setInterval(actionCallBack,delayInMs))
This lab introduces two sources of delay in treatment, resulting in an asynchronous thread : http requests and file access.
To do both tasks, you can use the FETCH API. This API uses a Promises. The Promise object represents the eventual completion (or failure) of an asynchronous operation and its resulting value.
You are asked to create a web page containing a html table of 10 random users (just the first name and last name). Each row will be filled with the response to a distinct http request to the randomUser API. Each request should be launched with a 1 second delay.
Hints :
  • first, make a single call to the randomUser API and output the result to the console. What is the form of the response? How can you extract the desired data (first name and last name)(look at the FETCH API)?
  • Then, repeat the call every seconds, ten times.
  • Finally, modify your code so that the output is displayed in a html table, updated when a http response is received.
